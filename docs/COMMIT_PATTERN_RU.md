# Паттерн Коммитов

Используем формат:

```text
<type>(<scope>): <subject>
```

Примеры:

- `feat(bot): добавить карточку заявки с QR`
- `fix(agent): остановить inflight-claim после паузы`
- `refactor(db): перевести active_orders на soft-close`
- `docs(tz): обновить ограничения sprint 1`

## Type

- `feat` — новая функциональность
- `fix` — исправление бага
- `refactor` — изменение структуры без новой функциональности
- `perf` — ускорение и оптимизация
- `test` — тесты
- `docs` — документация
- `chore` — сервисные изменения
- `ci` — CI/CD
- `build` — зависимости/сборка

## Scope

- `bot`, `agent`, `api`, `ws`, `db`, `infra`, `docs`, `tests`

## Правила

- Тема коммита: до 72 символов.
- Без точки в конце.
- Один коммит — одна логическая цель.
- Перед коммитом: `pytest` и базовая ручная проверка сценария.
